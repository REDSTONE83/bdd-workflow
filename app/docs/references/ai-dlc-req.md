# AI DLC Portal 구축 요구사항 정의서

---

## 1. 개요

### 1.1 목적
본 문서는 AI DLC(AI Development Lifecycle) Portal 구축을 위한 요구사항을 정의합니다. AI DLC Portal은 AI Agent 기반 개발을 조직 프로세스에 연결하는 통합 플랫폼으로, 개인 생산성 향상을 넘어 조직 차원의 안전한 개발 체계로 확장하기 위한 목적으로 구축됩니다.

### 1.2 범위
- AI DLC Portal 서비스 구축
- Skills Set 배포 및 공유 체계
- 산출물 관리 및 추적성 확보
- 변경 영향 분석 기능
- 권한 관리 및 협업 기능

---

## 2. 배경 및 필요성

### 2.1 현황 및 문제점

| 문제점 | 설명 |
|--------|------|
| 산출물 분산 | 산출물이 개인 PC, 브랜치, 로컬 워크스페이스에 흩어져 있음 |
| 추적성 부족 | Agent가 어떤 근거로 코드를 바꿨는지 추적 어려움 |
| 의미 관계 관리 한계 | Git은 코드 변경 관리에 강하지만 산출물 간 의미 관계 관리에는 부족 |
| 권한 제어 편중 | 권한 제어가 저장소 권한에 치우쳐 있음 |
| 영향 분석 의존 | 변경 영향 분석이 개발자 경험에 의존 |
| 보안 리스크 분산 | 보안·오픈소스·공급망 리스크가 로컬별로 다르게 처리됨 |

### 2.2 필요성

```
현재 AI DLC의 Ref. 제공만으로 고객 만족이 쉽지 않음
제안시 Ref. 만 제공하는 방식이 경쟁력이 떨어짐
지금 Ref. 방식으로는 개인 프로젝트/과제 수행은 가능해 보이나, 
팀 단위 작업 수행에 대한 의문이 우선적으로 발생함
```

## 8. 공유 서비스 역할 정의

### 8.1 핵심 역할

| 역할 | 설명 |
|------|------|
| 산출물 허브 | 요구사항, AC, 설계 메모, API, 테스트 시나리오, Agent 작업 로그, 코드 변경 요약을 중앙 등록 |
| 추적성 관리 | "이 코드가 어떤 요구사항과 테스트를 근거로 변경되었는가?"를 연결 |
| 변경 영향 분석 | 코드 변경 시 관련 API, 화면, DB, 테스트, 공통 모듈, 의존 서비스를 자동으로 추정 |
| 승인·통제 게이트 | 영향도가 큰 변경, 보안 취약점, 표준 위반, 테스트 미통과 변경은 병합 전 승인 대상으로 올림 |

### 8.2 역할 분담

```
로컬 Agent: 실행자 (Executor)
공유 서비스: 통제자 (Controller)

이 구조가 되어야 AI Agent 개발이 개인 생산성 향상에서 끝나지 않고, 
조직 차원의 안전한 개발 체계로 확장됩니다.
```
