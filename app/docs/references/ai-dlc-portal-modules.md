# AI DLC Portal - 상위 기능 모듈 구분

---

## 1. 모듈 구분 개요

AI DLC Portal은 "로컬 Agent는 실행자, Portal은 통제자(Control Plane)"의 방향에 따라 5개 상위 모듈로 구분합니다.

| # | 모듈명 | 핵심 역할 | 대응 요구사항 영역 |
|---|--------|-----------|-------------------|
| 1 | Skills Manager | Skill Set 배포/커스터마이징/재공유 | FUNC-001 ~ FUNC-006 |
| 2 | Workspace & Artifact Hub | 산출물 중앙 저장/관리/공유 | FUNC-011 ~ FUNC-020 |
| 3 | Traceability & Impact Analysis | Ontology 기반 추적성/변경영향 분석 | FUNC-021 ~ FUNC-025 |
| 4 | Review & Governance | 검토/승인/의사결정 이력 관리 | FUNC-031 ~ FUNC-036 |
| 5 | Administration | 사용자/권한/Tenant/인프라 관리 | FUNC-041 ~ FUNC-045 |

---

## 2. 모듈 상세 정의

### 2.1 Skills Manager

> "Skills Set 배포 공유" 역할 — AI Agent가 사용할 Skills의 배포 및 관리

**핵심 기능:**
- Skill Set 등록/배포/버전관리
- 프로젝트 전달용 Skill 패키징
- Skill 커스터마이징 및 재공유
- Skill 적용 이력 및 결과물 연결
- Skill 검색/발견/카탈로그
- Skill 의존성 분석 및 시각화

**포함 요구사항:** FUNC-001 ~ FUNC-006

**기술 스택:** RDB, 파일 저장소

---

### 2.2 Workspace & Artifact Hub

> "산출물 허브" 역할 — 로컬 Agent가 생성한 산출물을 중앙에서 수집, 저장, 공유

**핵심 기능:**
- 프로젝트별 작업 공간 관리 (Tenant → Project → Workspace 계층)
- 산출물 등록/저장/버전관리 (요구사항, AC, 설계, API, 테스트, Agent 로그, 코드 변경 요약)
- 팀원 간 산출물 공유 및 조회
- 파일 업로드/다운로드
- 산출물 체계/구조 정의 (템플릿 기반)
- GitLab/Repository 연계를 통한 코드 산출물 수집
- 산출물 권한 관리 (조회/편집/삭제/승인)

**포함 요구사항:** FUNC-011 ~ FUNC-020

**기술 스택:** RDB(PostgreSQL), 파일 저장소, Git 연동 API

---

### 2.3 Traceability & Impact Analysis

> "추적성 관리 + 변경 영향 분석" 역할 — 산출물 간 의미 관계를 분석하고 가시화

**핵심 기능:**
- Ontology 모델 정의 및 관리 (요구사항 → AC → 테스트 → 코드 → 배포 영향)
- 산출물 간 관계 자동/수동 연결
- Graph DB 기반 가시화 (네트워크 그래프)
- 변경 영향도 자동 추정 (관련 API, 화면, DB, 테스트, 공통 모듈)
- 정합성 점검 및 이슈 발견
- 요구사항 추적성 매트릭스 및 리포트

**포함 요구사항:** FUNC-021 ~ FUNC-025

**기술 스택:** Graph DB(Neo4j), Ontology 엔진

---

### 2.4 Review & Governance

> "승인·통제 게이트" 역할 — 산출물 변경에 대한 검토, 승인, 통제

**핵심 기능:**
- 산출물 검토 워크플로우 (제출 → 검토 → 승인/반려)
- 의사결정 이력 및 근거 기록 (프롬프트, 참조 산출물, 검증 결과)
- 변경 승인 게이트 (영향도 기반 자동 분류)
- 보안/표준 위반 경고 및 차단
- PR 연동 검토 (GitHub/GitLab PR에 분석 결과 주입)
- 알림 관리 (이메일, 시스템 내 알림)
- 감사 로그 (Audit Trail)

**포함 요구사항:** FUNC-031 ~ FUNC-036

**기술 스택:** Workflow 엔진, 알림 서비스

---

### 2.5 Administration

> "인프라 및 운영 관리" 역할 — 시스템 전체의 기반 설정 및 통제

**핵심 기능:**
- 사용자/조직/역할 관리 (RBAC)
- 산출물/기능/도메인별 권한 정책
- Tenant 관리 (SaaS 모드)
- 프로젝트 관리
- Repository/MCP 연동 설정
- 시스템 모니터링 및 로그
- Agent 토큰 관리
- On-Site / SaaS 배포 모드 지원

**포함 요구사항:** FUNC-041 ~ FUNC-054

**기술 스택:** RDB, 인증 서비스, MCP 서버

## 3. 모듈 간 관계

```
┌─────────────────────────────────────────────────┐
│              Administration                      │
│        (사용자, 권한, Tenant, 설정)               │
└──────────────┬──────────────────────────────────┘
               │ 권한/설정
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌─────────┐ ┌──────────┐ ┌────────────┐
│ Skills  │ │Workspace │ │  Review &  │
│ Manager │ │& Artifact│ │ Governance │
│         │ │  Hub     │ │            │
└────┬────┘ └────┬─────┘ └─────┬──────┘
     │            │             │
     └──────┬─────┘             │
            ▼                   │
┌────────────────────────────┐  │
│  Traceability & Impact     │  │
│       Analysis             │◄─┘
│  (Ontology, Graph DB)      │
└────────────────────────────┘
```

**데이터 흐름:**
- **Skills Manager** → Skill 적용 시 **Workspace**에 산출물 생성
- **Workspace** → 산출물/변경이 **Traceability**로 유입되어 관계 분석
- **Traceability** → 영향도 분석 결과를 **Review & Governance**에 전달
- **Administration** → 전 모듈에 권한/설정/테넌트 컨텍스트 제공

## 4. MVP 기능 축약

핵심 가치 흐름(산출물 등록 → 관계 연결 → 승인)을 검증할 수 있는 최소 기능으로 구분합니다.

### 4.1 Phase 1 (MVP) — "산출물이 중앙에 모여, 관계가 보이고, 승인이 됨"

> 프로젝트별 구분은 Phase 2로 이관. MVP는 단일 워크스페이스로 핵심 흐름 검증.
> 데이터 모델에는 `projectId`를预留하여 Phase 2에서 UI 필터링만 추가하면 확장 가능.

| 모듈 | 포함 기능 | 화면 수 |
|------|-----------|---------|
| Auth | 로그인, 사용자 정보 | 1 |
| 산출물 Hub | 등록, 조회/검색, 수정/삭제, 버전 이력, 폴더 구조 | 3 |
| 관계 연결 | 수동 관계 등록, 관계 그래프(기본) | 1 |
| 승인 | 승인 요청/처리, 이력 조회 | 2 |
| Admin | 사용자/역할 관리, 시스템 설정 | 2 |
| **합계** | | **9 화면** |

### 4.2 Phase 2 — "Skills가 관리되고, 프로젝트별로 구분됨"

| 모듈 | 포함 기능 |
|------|-----------|
| Skills Manager | 카탈로그, 패키지 생성/다운로드, 버전 관리 |
| 프로젝트 관리 | 프로젝트 생성/목록/멤버 관리, 프로젝트별 데이터 격리 |
| 대시보드 | 프로젝트 대시보드, 승인 현황, 이슈 현황 |

### 4.3 Phase 3 — "AI가 분석하고, Agent가 연동됨"

| 모듈 | 포함 기능 |
|------|-----------|
| 자동 분석 | 관계 자동 추출, 변경 영향도 분석, 정합성 검증 |
| Agent 연동 | 토큰 관리, MCP 연동, 자동 산출물 등록 |
| 알림/워크플로우 | 고급 워크플로우, 알림 템플릿, SLA 관리 |

### 4.4 Phase 4 — "조직 단위로 확장되고, 통제가 강화됨"

| 모듈 | 포함 기능 |
|------|-----------|
| AI Chat | Portal 내 AI 채팅, Skills 실행, 대화 이력 |
| 고급 분석 | Graph DB 고급 시각화, 추적성 커버리지, 리포트 |
| Tenant 관리 | SaaS 모드, 멀티 테넌트 |
