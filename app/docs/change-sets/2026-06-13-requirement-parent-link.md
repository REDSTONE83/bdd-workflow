# Change Set: 2026-06-13 요건 상위 관계 명시

상태: 진행중
요청일: 2026-06-13
변경 유형: 수정
영향 요건: REQ-016, REQ-017, REQ-018, REQ-019, REQ-020, REQ-022, REQ-023, REQ-024, REQ-025, REQ-027
논의 상태: 없음

## 요청 요약

- 하네스 요건 카드 표준에 추가한 `상위 요건` 헤더를 애플리케이션 하위 원자 요건 카드에 기입한다.
- 하네스 도구/표준 변경은 [2026-06-13 하네스 UI fixture와 실데이터 대응 보강](../../../harness/docs/change-sets/2026-06-13-harness-ui-data-correspondence.md)에서 추적한다.

## 작업 범위

- 카테고리 상위 요건 REQ-015의 하위 원자 요건 REQ-016~REQ-020에 `상위 요건: REQ-015`를 기입한다.
- 할 일 상위 요건 REQ-021의 하위 원자 요건 REQ-022, REQ-023, REQ-024, REQ-025, REQ-027에 `상위 요건: REQ-021`을 기입한다.

## 제외 범위

- REQ-026(카테고리 삭제 시 할 일 연결 해제)에는 `상위 요건`을 기입하지 않는다. 할 일 관리 기능의 생명주기 동작이 아니라 카테고리 삭제 정책의 부수효과라 REQ-021의 하위가 아니다. REQ-021과의 관계는 기존 `관련 요건`으로 유지한다.
- 독립 원자 요건(REQ-001, REQ-004, REQ-005, REQ-011)과 대체됨/구현 슬라이스 카드에는 `상위 요건`을 기입하지 않는다.
- 카드 범위, 수용 기준, 승인 상태 변경 없음. 헤더 메타데이터만 추가한다.

## 완료 조건

- REQ-016~020의 추적 상태에 `parentRequirementIds: [REQ-015]`가 들어가고, REQ-015 추적 상태에 `childRequirementIds`로 다섯 요건이 역산되어 들어간다.
- REQ-022/023/024/025/027의 추적 상태에 `parentRequirementIds: [REQ-021]`이 들어가고, REQ-021 추적 상태에 `childRequirementIds`로 다섯 요건이 역산되어 들어간다.
- `npm run app:validate`가 BLUE를 유지한다(헤더 추가가 카드 검사를 깨지 않는다).

## 검증 명령

- `npm run app:trace -- --requirement REQ-021`
- `npm run app:trace -- --requirement REQ-015`
- `npm run app:validate`

## 결정 로그

- 2026-06-13: 하위 원자 요건은 부모를 `상위 요건` 헤더로 명시 선언한다. 계층을 휴리스틱으로 파생하지 않기로 한 하네스 결정에 맞춘다.
- 2026-06-13: REQ-026은 의도적으로 부모를 두지 않는다. 카테고리 삭제(REQ-019)와 할 일 영역에 걸친 정책이라 어느 한 상위 요건의 생명주기 하위로 귀속시키지 않고, 교차 관계는 `관련 요건`으로 표현한다.
