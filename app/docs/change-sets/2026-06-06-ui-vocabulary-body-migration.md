# Change Set: 2026-06-06 UI 어휘 본문 정규화

상태: 진행중
요청일: 2026-06-06
변경 유형: 마이그레이션, 수정
영향 요건: REQ-011, REQ-017, REQ-018, REQ-019, REQ-022, REQ-024, REQ-025
논의 상태: 없음

## 요청 요약

- 하네스 UI 어휘 표준(`harness/docs/change-sets/2026-06-06-ui-component-terminology-standard.md`)이 정한 정규 명칭에 맞춰, application 요건 카드 본문의 `모달`/`다이얼로그` 별칭을 정규 명칭으로 치환한다.
- 치환 후 본문은 하네스가 새로 등록한 terminology ban(`다이얼로그`/`모달`/`팝업`/`확인창`)을 위반하지 않는다.

## 작업 범위

- 생성·수정 입력 화면을 가리키는 표현은 `입력 대화상자`로 치환한다.
  - REQ-017(카테고리 생성), REQ-018(카테고리 수정), REQ-022(할 일 생성), REQ-024(할 일 수정)의 "생성/수정 모달".
- 삭제·로그아웃 확인 화면을 가리키는 표현은 `확인 대화상자`로 치환한다.
  - REQ-019(카테고리 삭제), REQ-025(할 일 삭제)의 "삭제 확인 모달", REQ-011의 "로그아웃 확인 다이얼로그".
- 치환은 `범위`, `수용 기준`, `의사결정 로그`, Skeleton/BDD 테스트 리뷰 이력 등 카드 본문 전체를 대상으로 한다.
- 이번 대상 카드에서 별칭은 `범위`·`의사결정 로그`·Skeleton 이력에만 있었고 `수용 기준`·`.feature`·테스트 `Covers`에는 없었다. 따라서 AC/시나리오/테스트 추적 문자열 변경은 발생하지 않는다. (일반 원칙: 치환으로 AC 문장이 바뀌면 대응하는 `.feature` `Covers:`, FE BDD `Covers`, `@Covers`도 같은 작업에서 갱신한다.)

## 제외 범위

- 하네스 표준·deny-list·ban 게이트 구현. (하네스 Change Set이 담당)
- 코드 식별자/컴포넌트 API/파일명 변경. 화면 구현의 영문 `Dialog`/`modal` 등은 바꾸지 않는다.
- `체크박스`(REQ-027)는 이미 정규 명칭이라 치환 대상이 아니다.
- UI 어휘 정규 명칭 자체의 추가·변경.

## 완료 조건

- REQ-011·017·018·019·022·024·025 본문에 `모달`/`다이얼로그`/`팝업`/`확인창`이 남지 않는다.
- 별칭이 AC 문장에 없었으므로 `.feature`/`@Covers`/FE `Covers` 변경 없이 추적 매칭이 유지된다.
- `npm run app:validate`가 통과한다(terminology `BAN_VIOLATION` 0건 포함).

## 검증 명령

- `npm run app:validate`
- `npm run app:trace -- --requirement REQ-019`
- `node harness/tools/terminology.mjs validate --strict`

## 결정 로그

- `모달`은 입력 화면이면 `입력 대화상자`, 확인 화면이면 `확인 대화상자`로 의미를 구분해 치환한다. 단순 일괄 치환하지 않는다.
- 영문 코드 식별자(`Dialog`, `modal` prop 등)는 화면 구현 계약이므로 본 치환 대상에서 제외한다.
- 이번 대상 카드는 별칭이 AC 밖(`범위`/`의사결정 로그`/Skeleton 이력)에만 있어 실제 `@Covers`/`Covers` 추적 문자열 변경은 없었다. (일반 원칙: AC가 바뀌면 추적 문자열도 같은 작업에서 갱신해 RED 회귀를 막는다.)
- 본 Change Set은 하네스 UI 어휘 표준 도입을 전제로 하며, 표준/ban이 적용되기 전에는 단독으로 완료하지 않는다.
