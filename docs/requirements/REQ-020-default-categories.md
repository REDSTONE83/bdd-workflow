# 요건 카드

요건 ID: REQ-020
제목: 신규 사용자 기본 카테고리 제공
우선순위: 중간
상태: 검토중
요건 종류: 기능
명세 역할: 원자 요건
대상 시스템: application
제품 영역: category
품질 속성: usability
검증 수준: acceptance
관련 요건: REQ-015, REQ-001
대체 요건: 없음

## 사용자/목적

신규 사용자는 가입 직후 업무, 개인, 기타 기본 카테고리를 받아 빈 분류 상태에서 시작하지 않아야 한다.

## 범위

- 신규 사용자가 가입하면 시스템 기본 카테고리 3개가 자동 생성된다.
- 기본 카테고리는 업무, 개인, 기타 순서로 준비된다.
- 기본 카테고리는 일반 카테고리와 동일하게 수정하거나 삭제할 수 있다.

## 표준 용어

- category.category
- category.default
- category.name
- category.color
- category.displayOrder

## 제외 범위

- 기본 카테고리에 대한 별도 보호 정책
- 사용자별 기본 카테고리 템플릿 설정
- 가입 화면 동작 변경

## 수용 기준

- (API) 신규 사용자가 가입하면 업무(#2563EB), 개인(#16A34A), 기타(#6B7280) 세 개의 기본 카테고리가 준비된다

## 의사결정 로그

- 결정일: 2026-05-21
  결정: 신규 사용자 가입 시 시스템 기본 카테고리를 자동 생성한다.
  이유: 가입 직후 빈 카테고리 상태에서 분류 흐름을 막지 않는다.
  결정자: Product Owner
  영향: REQ-001 가입 흐름은 가입 성공 후 동일 트랜잭션에서 기본 시드를 생성한다.

- 결정일: 2026-05-21
  결정: 시스템 기본 카테고리 시드는 `업무`(#2563EB, 1024), `개인`(#16A34A, 2048), `기타`(#6B7280, 3072) 3개로 고정한다.
  이유: 가입 직후 정리 부담을 늘리지 않으면서 일반적 분류 흐름을 커버한다.
  결정자: Product Owner
  영향: 시드는 일반 카테고리와 동일한 권한 정책을 따른다.

- 결정일: 2026-05-21
  결정: 사용자는 시스템 기본 카테고리도 일반 카테고리와 동일하게 수정·삭제할 수 있다.
  이유: 기본 카테고리에만 별도 보호 규칙을 두면 사용자가 분류 체계를 자기 맥락에 맞게 정리하지 못한다.
  결정자: Product Owner
  영향: 별도 system default 보호 정책을 두지 않는다.
