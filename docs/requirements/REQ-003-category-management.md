# 요건 카드

요건 ID: REQ-003
제목: 할 일 카테고리 관리
우선순위: 중간
상태: 승인

## 사용자/목적

로그인 사용자는 자신의 할 일을 분류하기 위한 카테고리를 생성, 조회, 수정, 삭제할 수 있어야 한다. 또한 신규 사용자는 가입 직후 사용할 수 있는 시스템 기본 카테고리를 받는다.

## 범위

- 사용자는 이름, 색상, 설명, 정렬 순서를 입력해 카테고리를 생성한다.
- 이름은 앞뒤 공백을 제거한 뒤 저장하고 중복 판단에 사용한다.
- 정렬 순서를 입력하지 않으면 본인 카테고리의 최대값에 1024를 더한 값으로 자동 할당한다.
- 사용자는 자신의 카테고리 목록을 `displayOrder` 오름차순, 동률이면 `id` 오름차순으로 조회한다.
- 사용자는 자신의 카테고리 내용을 부분 수정한다. 요청에 포함된 필드만 변경하고 누락된 필드는 기존 값을 유지하며, `color` 또는 `description`을 명시적으로 `null`로 보내면 값을 비운다. 수정 시 포함된 필드는 생성과 동일한 검증 규칙(이름 trim·길이, 설명 길이, 색상 형식, 이름 중복)을 통과해야 한다.
- 사용자는 자신의 카테고리를 영구 삭제한다.
- 같은 사용자 안에서 카테고리 이름은 유일해야 한다.
- 신규 사용자가 가입할 때 시스템 기본 카테고리 3개가 자동 생성된다.
- 인증된 사용자는 요청 헤더 `X-User-Id`로 식별한다.

- 할 일과 카테고리의 연결 (REQ-002에서 다룬다)
- 다른 사용자와의 카테고리 공유
- 카테고리 정렬 정책의 자동 재배치 (예: 삭제 시 displayOrder gap 메우기)
- 색상에 대한 사용자 정의 팔레트 관리
- 시스템 기본 카테고리에 대한 별도 보호 (사용자는 기본 카테고리도 자유롭게 수정/삭제할 수 있다)
- 검색, 필터, 페이지네이션
- 정식 인증 방식 (세션, JWT 등)

## 수용 기준

- 유효한 정보이면 카테고리가 생성된다
- 이름은 앞뒤 공백이 제거되어 저장된다
- 이름이 비어 있거나 공백만 입력하면 카테고리 생성이 거절된다
- 이름이 50자를 초과하면 카테고리 생성이 거절된다
- 설명이 500자를 초과하면 카테고리 생성이 거절된다
- 색상 형식이 잘못되면 카테고리 생성이 거절된다
- 같은 사용자가 이미 등록한 이름이면 카테고리 생성이 거절된다
- 정렬 순서를 정하지 않으면 본인 카테고리 목록의 가장 뒤에 추가된다
- 본인의 카테고리는 정해진 정렬 순서대로 보이며, 같은 순서면 먼저 등록한 카테고리가 위로 정렬되어 보인다
- 신규 사용자가 가입하면 업무(#2563EB), 개인(#16A34A), 기타(#6B7280) 세 개의 기본 카테고리가 준비된다
- 수정 시 입력한 항목만 변경되고 입력하지 않은 항목은 기존 값을 유지한다
- 수정 시 색상을 명시적으로 비우면 색상이 지워진다
- 수정 시 설명을 명시적으로 비우면 설명이 지워진다
- 수정 시 이름이 비어 있거나 공백만 입력하면 수정이 거절된다
- 수정 시 이름이 50자를 초과하면 수정이 거절된다
- 수정 시 설명이 500자를 초과하면 수정이 거절된다
- 수정 시 색상 형식이 잘못되면 수정이 거절된다
- 수정 시 같은 사용자 안에서 이미 사용 중인 이름으로 바꾸려 하면 수정이 거절된다
- 존재하지 않는 카테고리를 수정하려 하면 거절된다
- 다른 사용자의 카테고리를 수정하려 하면 거절된다
- 본인의 카테고리를 영구 삭제할 수 있다
- 존재하지 않는 카테고리를 삭제하려 하면 거절된다
- 다른 사용자의 카테고리를 삭제하려 하면 거절된다
