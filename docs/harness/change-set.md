# Change Set 운영 가이드

Change Set은 사용자 요청을 처리하기 위해 함께 바꾸는 REQ 집합이다. 영구 명세 원천이 아니며, 제품 또는 하네스 명세의 원천은 계속 요건 카드다.

Change Set은 별도 사람이 관리하는 ID를 만들지 않는다. `docs/change-sets/YYYY-MM-DD-slug.md` 파일 경로가 작업 묶음의 identity이며, 하네스 리포트는 이 파일 경로를 기준으로 생성된다.

## 역할 분리

- 사용자 요청 = Change Set
- REQ = 명세 원천
- AC = 기계 검증 기준
- Scenario = 업무 검토 언어
- Test = 실행 검증
- Report = 생성된 목록과 추적표

## 필수 헤더

```text
상태: 계획 | 진행중 | 완료 | 보류
요청일: YYYY-MM-DD
변경 유형: 신규 | 수정 | 분리 | 병합 | 대체 | 폐기 | 마이그레이션 | 하네스 개선
영향 요건: 없음 또는 REQ-001, REQ-002
논의 상태: 없음 또는 열린 논의 요약
```

`변경 유형`은 쉼표로 여러 개를 적을 수 있다. 허용값 밖의 임의 작업명은 쓰지 않는다. `영향 요건`에는 이번 작업이 직접 수정하거나 재검토해야 하는 REQ를 적는다. 영향 요건이 아직 확정되지 않았으면 `없음`으로 두고 `열린 논의`에 확정해야 할 기준을 적는다.

`논의 상태`는 `## 열린 논의` 섹션의 짧은 요약이다. 열린 논의가 없으면 `논의 상태: 없음`과 `## 열린 논의`의 `- 없음`을 함께 쓴다. 열린 논의가 있으면 헤더에는 한 줄 요약을, 섹션에는 실제 미해결 항목을 bullet로 적는다.

## 필수 섹션

- `## 요청 요약`
- `## 작업 범위`
- `## 제외 범위`
- `## 완료 조건`
- `## 검증 명령`
- `## 결정 로그`
- `## 열린 논의`

## 작성 기준

- 사용자 요청 하나가 항상 새 REQ가 되는 것은 아니다. 먼저 기존 REQ 수정인지, 여러 REQ를 함께 바꾸는 Change Set인지, 새 원자 요건/통합 요건/정책 요건/하네스 요건이 필요한지 판단한다.
- Change Set에는 작업 범위, 영향 REQ, 검증 명령, 완료 조건만 둔다. 제품 동작의 진실은 REQ 카드의 범위/AC/의사결정 로그에 둔다.
- Change Set의 `완료 조건`은 작업 완료 판정이고, REQ의 `수용 기준`은 명세 완료 판정이다. 둘을 섞지 않는다.
- API/화면/검사기/리포트처럼 구현 순서나 구현 표면 때문에 갈라진 카드는 독립 REQ로 유지하지 않고 canonical REQ로 병합한다.
- 큰 업무 카드는 사용자 목표와 생명주기 단위로 원자 요건을 나누고, 같은 기능에 포함된다는 사실은 `명세 역할: 상위 요건`인 REQ, 동일 `제품 영역`, 하위 카드의 `관련 요건`으로 표현한다.
- 기존 REQ를 분리/병합/대체하는 작업은 Change Set으로 먼저 추적하고, 사용자와 적정 단위를 논의한 뒤 카드 상태와 `관련 요건`/`대체 요건`을 갱신한다.
- 완료된 Change Set은 파일 경로가 identity이므로 다른 위치로 옮기지 않는다. 작업이 끝나면 `상태: 완료`, 중단된 작업은 `상태: 보류`로 바꾸고 리포트의 상태별 집계로 수명주기를 확인한다.

## 리포트

하네스는 Change Set 원본을 직접 판단하지 않고 다음 생성 산출물로 확인한다.

- `build/harness/indexes/change-sets.index.json`: Change Set 원본에서 헤더, 섹션, 영향 REQ를 추출한 인덱스
- `build/harness/reports/change-set-report.md`
- `build/harness/reports/change-set-report.json`

Change Set 리포트는 `Change Set -> affected REQ -> 카드 스키마 상태 -> RED/GREEN/BLUE 상태`를 보여준다. 게이트 차단 기준은 아니며, 작업 범위와 마이그레이션 진행 상태를 확인하기 위한 리포트다.

Change Set 형식 검사는 report-only 경고로만 표시된다. `CHANGE-SET-*` 경고는 필수 헤더/섹션 누락, 허용값 외 상태/변경 유형, `논의 상태`와 `열린 논의` 불일치, 존재하지 않는 영향 REQ를 알려 주지만 `validateHarness`의 8개 게이트 카테고리에는 포함되지 않는다.
