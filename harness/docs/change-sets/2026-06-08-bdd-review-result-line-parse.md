# Change Set: 2026-06-08 BDD 리뷰 결과 라인 파싱

상태: 완료
요청일: 2026-06-08
변경 유형: 하네스 개선, 표준 개정
영향 요건: REQ-028
논의 상태: 없음

## 요청 요약

- PR #9 후속으로 승인 리뷰 영역의 `미완료` 단어 검사를 결과 라인 기반 구조 검사로 개선한다.
- 도메인 문장이나 Skeleton 승인 이력의 `미완료` 표현이 승인 게이트 false positive를 만들지 않게 한다.

## 작업 범위

- `BDD 테스트 리뷰` 파서가 최신 `결과:` 라인을 추출하도록 개선한다.
- 승인 카드의 BDD 리뷰 승인/미완료 판정을 최신 결과 라인 기준으로 변경한다.
- REQ-028 AC, 시나리오, 하네스 self-test를 결과 라인 기준으로 보강한다.
- 요건 카드 표준, 작성 가이드, 템플릿, 데이터 계약, 룰 설명을 같은 기준으로 갱신한다.

## 제외 범위

- 과거 리뷰 이력을 자동 정리하거나 삭제하지 않는다.
- `결과:` 상태값의 전체 enum을 별도 필드로 강제하지 않는다.
- 애플리케이션 요건 카드 상태를 일괄 변경하지 않는다.

## 완료 조건

- `BDD 테스트 리뷰`의 자유 텍스트 `미완료`는 승인 차단 사유가 아니다.
- 최신 `결과:` 라인이 `미완료`인 승인 카드는 `CARD-APPROVAL-BDD-INCOMPLETE`로 차단된다.
- 최신 `결과:` 라인이 `승인`인 승인 카드는 BDD 리뷰 승인 검사에서 통과한다.
- 관련 tool-test, self-test, harness validate가 통과한다.

## 검증 명령

- `npm run harness:tool-test`
- `npm run harness:self-test`
- `npm run harness:validate`

## 결정 로그

- 2026-06-08: 승인 리뷰 상태는 `BDD 테스트 리뷰` 섹션의 최신 `결과:` 라인으로 판정한다. 자유 텍스트와 `Skeleton 결과:`는 테스트 리뷰 상태로 보지 않는다.
