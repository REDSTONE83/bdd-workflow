# Change Set: 2026-06-10 입력 대화상자 어휘 표준

상태: 완료
요청일: 2026-06-10
변경 유형: 표준 개정
영향 요건: 없음
논의 상태: 없음

## 요청 요약

- 생성·수정 입력 팝업 UI를 `대화상자`만으로 표현할 때 의미가 넓어질 수 있으므로, 정규 명칭을 `입력 대화상자`로 개정한다.

## 작업 범위

- UI 컴포넌트 어휘 표준의 생성·수정 입력 UI 정규 명칭을 기존 입력 UI 명칭에서 `입력 대화상자`로 바꾼다.
- terminology 사전의 `ui.dialog` 허용 표현에 `입력 대화상자`를 추가한다.
- 기존 입력 UI 명칭은 더 이상 허용 표현으로 남기지 않고, 신규·기존 문서 모두 `입력 대화상자`를 쓰도록 명시한다.
- 기존 입력 UI 명칭을 terminology 사전 ban에 포함한다.
- 요건 작성 가이드의 예시와 설명을 새 정규 명칭에 맞춘다.

## 제외 범위

- 카드 `## 표준 용어`에 `ui.formDialog` 같은 UI 컴포넌트 원자 키를 허용하는 정책 변경.
- 기존 요건 카드 AC와 FE BDD `Covers` 문장 일괄 변경.

## 완료 조건

- UI 어휘 표준에서 입력 팝업 UI의 정규 명칭이 `입력 대화상자`로 표시된다.
- `다이얼로그`/`모달`/`팝업` 별칭은 계속 `대화상자` 계열로 정규화된다.
- 기존 입력 UI 명칭은 본문 ban으로 차단된다.
- 작성 가이드는 확인 목적이면 `확인 대화상자`, 생성·수정 입력 목적이면 `입력 대화상자`를 쓰도록 안내한다.

## 검증 명령

- `npm run harness:self-test`

## 결정 로그

- 2026-06-10: 생성·수정 입력 UI의 정규 명칭은 `입력 대화상자`로 둔다. 기존 입력 UI 명칭은 허용하지 않는다.
- 2026-06-10: `ui.formDialog` 같은 컴포넌트 원자 키는 계속 카드 `## 표준 용어` 금지 대상이다. 이번 변경은 본문 작성 어휘만 바꾼다.
