# UI 컴포넌트 어휘 표준

이 표준은 요건 본문·수용 기준·시나리오·프런트엔드 표준 문서에서 UI 컴포넌트/위젯을 가리킬 때 쓰는 **문서 작성 어휘**를 고정한다. 같은 컴포넌트를 `다이얼로그`/`모달`/`팝업`처럼 섞어 부르지 않게 하는 것이 목적이다.

## 추적용 표준 용어와의 분리

용어는 두 레이어로 나뉜다.

- **추적용 표준 용어**(`## 표준 용어` 섹션, `domains/*.json`): 카드의 비즈니스 의미에 핵심적인 업무/정책/품질/화면 품질 단위. 카드별로 등록하고 추적한다.
- **문서 작성용 UI 어휘**(이 표준): UI 컴포넌트/위젯 원자. 카드 `## 표준 용어`에 등록하지 않고, 본문에서 정규 명칭으로만 쓴다.

`ui.appShell`, `ui.desktopViewport`, `ui.accessibilityCheck`처럼 화면 품질·플랫폼 의미를 가진 용어는 추적용 표준 용어이며 카드 `## 표준 용어`에 둘 수 있다. `ui.button`, `ui.dialog` 같은 컴포넌트 원자는 이 표준의 대상이다.

## 정규 명칭

| 정규 명칭 | 영문 | 피해야 할 별칭 | 사용 위치/비고 |
| --- | --- | --- | --- |
| 대화상자 | dialog | 다이얼로그, 모달, 팝업 | 화면 위에 겹쳐 띄우는 모드형 표시 영역 |
| 확인 대화상자 | confirm dialog | 확인창 | 삭제·로그아웃 등 동작 확인 |
| 입력 대화상자 | form dialog | 폼 대화상자 | 생성·수정 입력 |
| 체크박스 | checkbox | 체크 박스 | 켜짐/꺼짐 선택 컨트롤 |
| 텍스트 입력 | text input | 인풋 | 단일 행 문자열 입력 |
| 아이콘 버튼 | icon button | — | 아이콘만 표시하는 버튼 |
| 드롭다운 | dropdown | — | 펼침 선택 목록 |
| 탭 | tab | — | 화면 영역 전환 |
| 토글 | toggle | — | 두 상태 전환 스위치 |
| 도구 설명 | tooltip | 툴팁 | hover 시 보조 설명 |

정규 명칭 목록은 현재 화면 요건에서 실제 쓰이는 컴포넌트로 한정한다. 새 컴포넌트가 필요하면 이 표에 정규 명칭과 별칭을 증분 추가한다.

## 카드 `## 표준 용어`에 두지 않는다

UI 컴포넌트 원자 키는 카드 `## 표준 용어` 섹션에 둘 수 없다. 다음 키는 `validate-requirement-cards.mjs`의 deny-list로 차단되며 `CARD-TERM-UI-PRIMITIVE` 오류로 보고된다.

```text
ui.button  ui.input  ui.checkbox  ui.dialog  ui.formDialog  ui.confirmDialog
ui.tooltip  ui.modal  ui.dropdown  ui.tab  ui.toggle
```

이 판정은 `CARD-TERM-UNREGISTERED`보다 먼저 수행한다. 즉 "용어집에 등록하지 않은 누락"이 아니라 "잘못된 레이어에 둔 용어"로 안내한다. 해당 의미는 용어집에 등록하지 말고, 위 정규 명칭으로 본문/수용 기준/시나리오에 표현한다.

deny-list는 코드 상수가 단일 소스(SSOT)다. 이 문서는 그 목록을 설명·링크만 하며, 새 UI 컴포넌트는 코드 상수에 증분 추가한다.

## 본문 재발 방지(ban)

가장 자주 혼용되는 별칭은 terminology 사전 ban으로 등록되어 본문에서 `BAN_VIOLATION`으로 차단된다.

- `다이얼로그`, `모달`, `팝업`, `확인창`, `폼 대화상자` → `대화상자`(또는 `확인 대화상자`/`입력 대화상자`)
- `체크 박스` → `체크박스`
- `인풋` → `텍스트 입력`

사전에 UI 어휘 term(`ui.dialog`·`ui.checkbox`·`ui.input`)을 두는 것은 **본문 ban/allow surface 통제** 용도다. 사전에 있어도 카드 `## 표준 용어` 사용은 위 deny-list가 막는다. 즉 **사전 등록 ≠ 카드 표준 용어 허용**이다.

표의 나머지 별칭(`툴팁` 등)은 게이트로 차단하지 않고 권고로 둔다. 혼용이 반복되면 사전 ban으로 승격한다.

## 코드 식별자는 대상이 아니다

이 표준은 **문서 작성 어휘**만 통제한다. 프런트엔드 컴포넌트 API, shadcn/ui 파일명, prop 이름 같은 코드 식별자(`Dialog`, `dialog.tsx`, `modal` 등 영문)는 ban 대상이 아니며 그대로 사용한다. ban surface는 한국어 별칭 본문 표현만 매치한다.
